## 1. Design Principles

### 1.1 Core Principles

| Principle | Description | Application |
|-----------|-------------|-------------|
| **Mobile-First** | Design for smallest screens first, enhance for larger | All components start at 320px, scale up |
| **Progressive Disclosure** | Reveal complexity gradually | Checkout shows one step at a time |
| **Conversion-Focused** | Every element serves the purchase journey | CTAs prominent, friction minimized |
| **Accessible by Default** | WCAG 2.1 AA compliance built-in | Focus states, color contrast, screen readers |
| **Brand Flexibility** | Support tenant customization without breaking UX | CSS variables, constrained color choices |
| **Performance-First** | Fast loads = higher conversion | Lazy loading, optimized images, minimal JS |

### 1.2 Design Philosophy

#### For Maya (Merchant)
- **Simplicity over features**: Dashboard should feel like a consumer app, not enterprise software
- **Visual feedback**: Every action should have immediate, visible confirmation
- **Guided workflows**: Onboarding and setup should feel like a conversation, not a form
- **Confidence building**: Show progress, celebrate milestones, provide helpful defaults

#### For Sam (Shopper)
- **Speed is trust**: Fast page loads signal a professional, trustworthy store
- **Thumb-friendly**: Primary actions in the bottom 1/3 of mobile screens
- **Clear pricing**: No hidden costs, transparent totals at every step
- **Easy recovery**: Back buttons, edit options, clear error messages

#### 3.2.2 Input

| Property | Description |
|----------|-------------|
| **Purpose** | Text input for forms |
| **Types** | `text`, `email`, `password`, `number`, `tel`, `search` |
| **States** | Default, Focus, Error, Disabled |

**Accessibility:**
- Always paired with `<Label>` via `htmlFor`
- Error state: `aria-invalid="true"`, `aria-describedby` for error message

**Theming Customization:**
- `--color-input` â†’ Background color
- `--color-border` â†’ Border color
- `--color-ring` â†’ Focus ring color
- `--radius-md` â†’ Border radius

```typescript
    <h3>{product.name}</h3>
    <div className="flex items-center gap-2">
      <span className="font-semibold">${price}</span>
      {hasDiscount && (
        <span className="line-through text-muted-foreground">
          ${compareAtPrice}
        </span>
      )}
    </div>
    <Button onClick={handleAddToCart}>Add to Cart</Button>
  </CardContent>

```

### 4.4 Customization Limits (Accessibility Guardrails)

To maintain WCAG 2.1 AA compliance, certain customizations are constrained:

| Property | Constraint | Reason |
|----------|------------|--------|
| **Primary color** | Must have 4.5:1 contrast with foreground | Text readability |
| **Body font size** | Minimum 16px (1rem) | Prevents iOS zoom, readability |
| **Focus ring** | Cannot be disabled | Keyboard navigation |
| **Button min-height** | 44px touch target | Mobile accessibility |
| **Link underlines** | Required on hover minimum | Distinguishable from text |
| **Error indicators** | Must include icon, not just color | Color blindness |

```typescript
// Validation function for theme customization
function validateThemeAccessibility(theme: TenantTheme): ValidationResult {
  const errors: string[] = [];
  
  // Check primary color contrast
  const primaryContrast = getContrastRatio(
    theme.colors.primary, 
    theme.colors.primaryForeground
  );
  if (primaryContrast < 4.5) {
    errors.push(`Primary color contrast (${primaryContrast.toFixed(2)}) is below 4.5:1`);
  }
  
  // Check body font size
  const baseFontSize = parseFloat(theme.typography.scale.base);
  if (baseFontSize < 1) { // Less than 1rem = 16px
    errors.push('Body font size must be at least 16px (1rem)');
  }
  
  // Check background/foreground contrast
  const bgContrast = getContrastRatio(
    theme.colors.background,
    theme.colors.foreground
  );
  if (bgContrast < 4.5) {
    errors.push(`Background/foreground contrast (${bgContrast.toFixed(2)}) is below 4.5:1`);
  }
  
  return {
    valid: errors.length === 0,
    errors
  };
}
```

### 6.5 Error Prevention Patterns

| Scenario | Prevention | Recovery |
|----------|------------|----------|
| **Invalid email** | Inline validation on blur | Clear error message, focus field |
| **Out of stock** | Disable add-to-cart, show message | "Notify when available" option |
| **Payment failure** | Pre-validate card format | Specific error, retry option |
| **Session timeout** | Auto-save cart to localStorage | Restore on return |
| **Network error** | Optimistic UI, retry logic | Toast with retry button |
| **Form abandonment** | Unsaved changes warning | "Save for later" option |
